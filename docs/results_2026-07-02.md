## Dataset

- Rows: 6,000
- Fraud rate: 9.08%
- Data source: synthetic, generated locally
- Split: 60% train, 20% validation, 20% test
- Target leakage check: target is not used as feature
- Duplicate transaction IDs: 0

## Interpretation

The MVP is tuned to catch most simulated fraud cases, accepting a high manual-review rate. This mirrors an early fraud-screening workflow where missed fraud is more expensive than reviewing legitimate transactions.

The model catches 92 of 109 fraud cases in the holdout test set. Precision is intentionally modest because the selected threshold prioritizes recall and operational loss reduction.

## Limitations

- The dataset is synthetic, so results should not be presented as real-world fraud performance.
- SHAP is available as an optional report, but the API still uses lightweight reason codes.
- XGBoost is available as an optional model, but RandomForest currently wins by validation operational cost.
- The public sample adapter is a smoke test, not a strong benchmark.
- A public real-world benchmark should be added before claiming production-level performance.
